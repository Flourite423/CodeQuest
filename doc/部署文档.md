## 一、项目概述

CodeQuest 是一个面向编程学习者的学习应用，包含：

| 端 | 技术栈 | 用途 | 默认端口 |
|---|--------|------|---------|
| **Backend** | Rust + Salvo + SQLx + PostgreSQL | REST API 服务 | 3001 |
| **Mobile** | Flutter 3.x (Web 构建) | 学习者客户端 | 8088 |
| **Admin** | Vue 3 + Element Plus (构建产物) | 管理后台 | 3000 |

## 四、数据库部署与迁移

### 4.1 安装 PostgreSQL

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install postgresql postgresql-contrib

# 启动服务
sudo systemctl start postgresql
sudo systemctl enable postgresql
```

### 4.2 创建数据库与用户

```bash
# 切换到 postgres 用户
sudo -u postgres psql

# 在 psql 中执行
CREATE DATABASE learning_app;
CREATE USER codequest WITH PASSWORD 'your-secure-password';
GRANT ALL PRIVILEGES ON DATABASE learning_app TO codequest;
\q
```

### 4.3 执行数据库迁移

**方式一：使用 sqlx-cli（推荐）**

```bash
# 安装 sqlx-cli
cargo install sqlx-cli --no-default-features --features native-tls,postgres

# 进入后端源码目录
cd backend

# 设置数据库 URL
export DATABASE_URL="postgres://codequest:your-secure-password@localhost/learning_app"

# 执行迁移
sqlx migrate run
```

**方式二：直接执行 SQL 文件**

```bash
# 进入项目根目录后，按顺序执行 migrations 目录下的 SQL 文件
sudo -u postgres psql -d learning_app -f backend/migrations/001_initial_schema.sql
sudo -u postgres psql -d learning_app -f backend/migrations/002_increase_refresh_token_length.sql
# ... 依次执行所有迁移文件
```

**方式三：自动迁移（生产环境启动时）**

后端支持启动时自动执行迁移，设置环境变量即可：

```bash
export APP__AUTO_RUN_MIGRATIONS="true"
```

启动后端服务后，程序会在启动阶段自动检测并执行未完成的迁移。

### 4.4 导入种子数据（可选）

```bash
# 导入初始测试数据（包含预设课程、挑战、账号等）
sudo -u postgres psql -d learning_app -f backend/seed_data.sql
```

通过环境变量也可在启动时自动导入：

```bash
export APP__SEED_DEV_ACCOUNTS="true"
```

### 4.5 验证数据库

```bash
# 登录数据库检查表结构
sudo -u postgres psql -d learning_app -c "\dt"

# 检查迁移记录
sudo -u postgres psql -d learning_app -c "SELECT * FROM _sqlx_migrations ORDER BY version;"
```

## 五、源文件部署（使用预编译程序包）

适用于已提供编译产物的场景，解压后直接配置运行，无需构建环境。

### 5.1 部署目录结构

```
/opt/codequest/
├── backend/
│   ├── learning-app-backend    # 可执行文件
│   └── .env                    # 环境变量文件（可选）
├── admin/                      # 管理后台静态文件
│   ├── index.html
│   └── assets/
├── mobile/                     # 移动端静态文件
│   ├── index.html
│   └── assets/
└── logs/                       # 日志目录
```

### 5.2 部署后端服务

```bash
# 解压源程序包
unzip CodeQuest-源程序.zip -d /opt/codequest

# 赋予执行权限
chmod +x /opt/codequest/backend/learning-app-backend

# 设置环境变量后启动
export APP__SERVER_ADDR="0.0.0.0:3001"
export APP__DATABASE_URL="postgres://codequest:your-secure-password@localhost/learning_app"
export APP__JWT_SECRET="your-jwt-secret-minimum-32-characters-long"
export APP__JWT_EXPIRATION="86400"
export APP__AUTO_RUN_MIGRATIONS="false"
export APP__SEED_DEV_ACCOUNTS="false"

# AI 配置（可选）
export APP__AI__PROVIDER="mock"    # 或 "deepseek"
export APP__AI__API_KEY=""
export APP__AI__MODEL="deepseek-chat"

# 启动服务
/opt/codequest/backend/learning-app-backend
```

#### 5.2.1 使用 systemd 管理（生产环境推荐）

创建服务文件 `/etc/systemd/system/codequest-backend.service`：

```ini
[Unit]
Description=CodeQuest Backend Service
After=network.target postgresql.service
Requires=postgresql.service

[Service]
Type=simple
User=codequest
WorkingDirectory=/opt/codequest/backend
ExecStart=/opt/codequest/backend/learning-app-backend
Restart=on-failure
RestartSec=5
LimitNOFILE=65536

# 环境变量
Environment="APP__SERVER_ADDR=0.0.0.0:3001"
Environment="APP__DATABASE_URL=postgres://codequest:your-secure-password@localhost/learning_app"
Environment="APP__JWT_SECRET=your-jwt-secret-minimum-32-characters-long"
Environment="APP__JWT_EXPIRATION=86400"
Environment="APP__AI__PROVIDER=mock"

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl enable codequest-backend
sudo systemctl start codequest-backend

# 查看状态
sudo systemctl status codequest-backend

# 查看日志
sudo journalctl -u codequest-backend -f
```

### 5.3 部署前端静态文件

#### 5.3.1 管理后台 (Admin)

将 `CodeQuest-Program/admin/` 目录下的所有文件复制到部署目录：

```bash
cp -r CodeQuest-Program/admin/* /opt/codequest/admin/
```

Nginx 配置：

```nginx
server {
    listen 3000;
    server_name admin.your-domain.com;

    root /opt/codequest/admin;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:3001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

#### 5.3.2 移动端 Web (Mobile)

```bash
cp -r CodeQuest-Program/mobile/* /opt/codequest/mobile/
```

Nginx 配置：

```nginx
server {
    listen 8088;
    server_name app.your-domain.com;

    root /opt/codequest/mobile;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:3001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

```

#### 5.3.3 Nginx 完整配置

```nginx
# /etc/nginx/sites-available/codequest

upstream codequest_backend {
    server 127.0.0.1:3001;
}

server {
    listen 80;
    server_name admin.your-domain.com;

    root /opt/codequest/admin;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://codequest_backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

server {
    listen 80;
    server_name app.your-domain.com;

    root /opt/codequest/mobile;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://codequest_backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/codequest /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

## 六、从源代码部署

适用于需要自行编译或定制构建的场景，部署前需准备完整的开发环境。

### 6.1 构建后端

```bash
# 进入后端目录
cd backend

# 安装 sqlx-cli（如尚未安装）
cargo install sqlx-cli --no-default-features --features native-tls,postgres

# 执行数据库迁移（需先配置 DATABASE_URL）
export DATABASE_URL="postgres://codequest:your-secure-password@localhost/learning_app"
sqlx migrate run

# 开发构建
cargo build

# 生产构建（优化体积和执行效率）
cargo build --release

# 运行测试（可选）
cargo test

# 代码检查（可选）
cargo clippy

# 编译产物
# 开发模式: target/debug/learning-app-backend
# 生产模式: target/release/learning-app-backend
```

### 6.2 构建管理后台

```bash
# 进入管理后台目录
cd admin

# 安装依赖
npm install

# 开发运行（带热更新）
npm run dev

# 生产构建
npm run build

# 代码检查（可选）
npm run lint

# 构建产物位于 dist/ 目录
```

### 6.3 构建移动端


### 6.4 部署构建产物

构建完成后，将产物部署到服务器：

```bash

# 部署管理后台
sudo cp -r admin/dist/* /opt/codequest/admin/

# 部署移动端
```

## 七、环境变量配置

### 7.1 后端环境变量

所有环境变量以 `APP__` 为前缀，使用双下划线表示层级：

| 变量名 | 必填 | 默认值 | 说明 |
|--------|------|--------|------|
| `APP__SERVER_ADDR` | 是 | `0.0.0.0:3001` | 服务监听地址 |
| `APP__DATABASE_URL` | 是 | - | PostgreSQL 连接字符串 |
| `APP__JWT_SECRET` | 是 | - | JWT 签名密钥（至少32位） |
| `APP__JWT_EXPIRATION` | 否 | `86400` | Token 过期时间（秒） |
| `APP__AUTO_RUN_MIGRATIONS` | 否 | `false` | 启动时自动执行迁移 |
| `APP__SEED_DEV_ACCOUNTS` | 否 | `false` | 启动时导入测试账号 |
| `APP__AI__PROVIDER` | 否 | `mock` | AI 提供商：mock/deepseek |
| `APP__AI__API_KEY` | 否 | - | DeepSeek API 密钥 |
| `APP__AI__MODEL` | 否 | `deepseek-chat` | AI 模型名称 |
| `APP__AI__TEMPERATURE` | 否 | `0.3` | 生成温度 |
| `APP__AI__MAX_TOKENS` | 否 | `500` | 最大 Token 数 |

### 7.2 配置文件

后端也支持通过 TOML 配置文件进行配置：

```toml
# config/default.toml
server_addr = "0.0.0.0:3001"
database_url = "postgres://user:pass@localhost/learning_app"
jwt_secret = "your-secret-key"
jwt_expiration = 86400

seed_dev_accounts = false

[ai]
```

## 九、常见问题

### 9.1 数据库连接失败

**症状**: 后端启动时报数据库连接错误

**解决**:
1. 检查 PostgreSQL 服务是否运行：`sudo systemctl status postgresql`
2. 检查数据库和用户是否存在
3. 检查连接字符串格式是否正确
4. 检查防火墙设置

### 9.2 跨域问题

**症状**: 前端无法调用 API

**解决**: 后端已内置 CORS 支持，确保前端请求的 `Origin` 在允许列表中。如需修改，请查看后端配置。

### 9.3 端口冲突

**症状**: 启动时报 "Address already in use"

**解决**:
```bash
# 查看端口占用
sudo lsof -i :3001

# 终止占用进程
sudo kill -9 <PID>
```

### 9.4 权限问题

**症状**: 后端无法绑定端口或读写文件

**解决**:
```bash
# 确保二进制文件有执行权限
chmod +x learning-app-backend

# 如需使用 80/443 端口，需 root 权限或使用 systemd
```

### 9.5 迁移失败

**症状**: `sqlx migrate run` 执行报错

**解决**:
1. 检查 `DATABASE_URL` 是否正确
2. 检查数据库用户是否有建表权限
3. 查看迁移文件是否有语法错误
4. 如迁移已部分执行，使用 `sqlx migrate revert` 回退后重试
