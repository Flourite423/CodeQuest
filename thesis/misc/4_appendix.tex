\chapter{AI 三级智能提示设计与 Prompt 示例}

为说明系统如何在编码练习场景中提供分层学习支持，本文将 AI 辅助提示划分为三级：第一级是\textbf{错误定位提示}，目标是帮助学习者理解“哪里出了问题”；第二级是\textbf{修正方向提示}，目标是帮助学习者理解“应该往哪个方向改”；第三级是\textbf{操作建议提示}，目标是帮助学习者在不直接获得完整答案的前提下，得到可以立即尝试的下一步操作。

三级提示的定位如表\ref{tab:ai-three-level-design}所示。

\begin{table}[htbp]
  \centering
  \caption{AI 三级智能提示设计}
  \label{tab:ai-three-level-design}
  \small
  \setlength{\tabcolsep}{4pt}
  \begin{tabular}{L{2.0cm}L{2.2cm}L{2.6cm}L{2.2cm}} 
    \toprule
    提示级别 & 设计目标 & 适用场景 & 输出边界 \\
    \midrule
    一级：定位 & 指出错误位置 & 首次提交报错 & 只说位置，不给代码 \\
    二级：方向 & 给出修正方向 & 已知错点但不会改 & 给原因和方向 \\
    三级：建议 & 提供操作步骤 & 多次尝试仍卡住 & 给 2--4 步建议 \\
    \bottomrule
  \end{tabular}
\end{table}

三级提示并不是三套完全独立的接口，而是同一 AI 辅助能力下的三层输出策略。移动客户端调用 \path{/learner/ai/help} 时沿用统一的请求结构；服务端或 AI 编排层根据当前的 request\_type、失败上下文、提交记录和提示次数，决定返回一级、二级还是三级风格的内容。输入信息包括题目要求 (prompt)、提交代码 (source\_code)、提交记录 (submission)、错误上下文 (error\_context)、请求类型 (request\_type) 和提示层级 (hint\_level)。

以下对三级提示分别说明其系统提示词、传给大模型的请求数据格式、以及大模型返回的结构化数据格式。

% ============ 一级提示 ============
\subsection*{一级提示：错误定位}

\subsubsection*{系统提示词}

发送给大语言模型的系统提示如下：

\begin{quote}
你是一名面向前端初学者的学习辅助教练。请根据题目要求、学习者提交的 HTML/CSS 代码和失败上下文，\textbf{只解释当前最可能出错的位置与现象}。

不要直接给出完整答案代码。

请使用简洁中文回答，包含以下三部分：
1) 当前错误最可能出现在哪一段结构或样式；
2) 它导致了什么可观察现象；
3) 学习者下一步应该先检查什么。
\end{quote}

\subsubsection*{请求数据格式（Input JSON）}

服务端调用大语言模型 API 时，将一份结构化请求体发送给模型。其核心字段示例如下：

\begin{quote}\small\ttfamily
\{
\ \ "model":\ "deepseek-chat",\\
\ \ "messages":\ [\\
\ \ \ \ \{\\
\ \ \ \ \ \ "role":\ "system",\\
\ \ \ \ \ \ "content":\ "<上述系统提示词>"\\
\ \ \ \ \},\\
\ \ \ \ \{\\
\ \ \ \ \ \ "role":\ "user",\\
\ \ \ \ \ \ "content":\ \{\\
\ \ \ \ \ \ \ \ "exercise\_prompt":\ "请使用 HTML 和 CSS\\
\ \ \ \ \ \ \ \ \ \ 实现 3 列卡片布局...",\\
\ \ \ \ \ \ \ \ "source\_code":\ "学习者提交的完整\\
\ \ \ \ \ \ \ \ \ \ HTML/CSS 源码",\\
\ \ \ \ \ \ \ \ "error\_context":\ \{\\
\ \ \ \ \ \ \ \ \ \ "failing\_case":\ "mobile\_single\_column",\\
\ \ \ \ \ \ \ \ \ \ "actual\_columns":\ 2\\
\ \ \ \ \ \ \ \ \},\\
\ \ \ \ \ \ \ \ "attempt\_no":\ 3,\\
\ \ \ \ \ \ \ \ "hint\_level":\ 1\\
\ \ \ \ \ \ \}\\
\ \ \ \ \}\\
\ \ ],\\
\ \ "temperature":\ 0.3,\\
\ \ "max\_tokens":\ 600\\
\}
\end{quote}

\subsubsection*{返回数据格式（Output JSON）}

大语言模型需以结构化 JSON 格式返回结果，由服务端解析后下发给移动客户端：

\begin{quote}\small\ttfamily
\{
\ \ "hint\_level":\ 1,\\
\ \ "summary":\ "窄屏断点下仍显示两列布局，疑似媒体查询\\
\ \ \ \ 未生效",\\
\ \ "error\_location":\ \{\\
\ \ \ \ "section":\ "CSS 媒体查询",\\
\ \ \ \ "selector":\ ".cards",\\
\ \ \ \ "property":\ "grid-template-columns"\\
\ \ \},\\
\ \ "observable\_symptom":\ "在视口宽度 375px 下，容器仍\\
\ \ \ \ 保持多列，说明媒体查询中的列定义未覆盖原有样式",\\
\ \ "next\_check":\ "请检查 @media 断点条件是否与测试场景\\
\ \ \ \ 一致，并确认媒体查询内部是否重写了 .cards 的\\
\ \ \ \ grid-template-columns"\\
\}
\end{quote}

一级提示适合首次失败或错误现象较明显的场景。返回字段 \texttt{error\_location} 直接指出 CSS 媒体查询中的选择器和属性，\texttt{next\_check} 引导学习者优先排查断点条件和样式覆盖逻辑。

% ============ 二级提示 ============
\subsection*{二级提示：修正方向}

\subsubsection*{系统提示词}

\begin{quote}
你是一名面向前端初学者的学习辅助教练。请在不直接给出完整代码答案的前提下，\textbf{解释问题产生的原因，并给出修正方向}。

请结合题目要求、提交代码和失败上下文，输出以下内容：
1) 失败最可能由哪一类原因导致；
2) 学习者应优先修改布局结构、选择器覆盖、断点条件还是属性值；
3) 给出 2 条不含完整答案的方向性建议。
\end{quote}

\subsubsection*{请求数据格式（Input JSON）}

\begin{quote}\small\ttfamily
\{
\ \ "model":\ "deepseek-chat",\\
\ \ "messages":\ [\\
\ \ \ \ \{\\
\ \ \ \ \ \ "role":\ "system",\\
\ \ \ \ \ \ "content":\ "<上述系统提示词>"\\
\ \ \ \ \},\\
\ \ \ \ \{\\
\ \ \ \ \ \ "role":\ "user",\\
\ \ \ \ \ \ "content":\ \{\\
\ \ \ \ \ \ \ \ "exercise\_prompt":\ "请使用 HTML 和 CSS\\
\ \ \ \ \ \ \ \ \ \ 实现 3 列卡片布局...",\\
\ \ \ \ \ \ \ \ "source\_code":\ "学习者提交的完整\\
\ \ \ \ \ \ \ \ \ \ HTML/CSS 源码",\\
\ \ \ \ \ \ \ \ "error\_context":\ \{\\
\ \ \ \ \ \ \ \ \ \ "failing\_case":\ "mobile\_single\_column",\\
\ \ \ \ \ \ \ \ \ \ "actual\_columns":\ 2\\
\ \ \ \ \ \ \ \},\\
\ \ \ \ \ \ \ \ "attempt\_no":\ 3,\\
\ \ \ \ \ \ \ \ "hint\_level":\ 2\\
\ \ \ \ \ \}\\
\ \ \ \ \}\\
\ \ ],\\
\ \ "temperature":\ 0.3,\\
\ \ "max\_tokens":\ 800\\
\}
\end{quote}

\subsubsection*{返回数据格式（Output JSON）}

\begin{quote}\small\ttfamily
\{
\ \ "hint\_level":\ 2,\\
\ \ "summary":\ "媒体查询断点未命中，导致移动端列布局\\
\ \ \ \ 未切换",\\
\ \ "root\_cause":\ \{\\
\ \ \ \ "category":\ "media\_query\_mismatch",\\
\ \ \ \ "detail":\ "媒体查询的断点宽度可能\\
\ \ \ \ \ \ 与测试视口场景不完全匹配；此外，媒体查询内部\\
\ \ \ \ \ \ 可能仅调整了间距而未重写列定义"\\
\ \ \},\\
\ \ "direction":\ \{\\
\ \ \ \ "priority":\ "媒体查询断点与属性覆盖",\\
\ \ \ \ "reason":\ "当前错误并非布局策略选择错误，\\
\ \ \ \ \ \ 而是断点条件下的样式没有按预期生效"\\
\ \ \},\\
\ \ "suggestions":\ [\\
\ \ \ \ "先确认断点条件是否覆盖到测试场景的视口宽度",\\
\ \ \ \ "在媒体查询内部显式重写容器的列定义"\\
\ \ ]\\
\}
\end{quote}

二级提示比一级更进一步：\texttt{root\_cause} 将现象归因到具体错误类别；\texttt{direction} 指出应优先修正断点条件而非调整布局算法；\texttt{suggestions} 给出方向性建议但不提供完整代码答案。

% ============ 三级提示 ============
\subsection*{三级提示：操作建议}

\subsubsection*{系统提示词}

\begin{quote}
你是一名面向前端初学者的学习辅助教练。学习者已经获得基础解释，但仍然没有修复问题。请输出\textbf{可执行的下一步操作建议}，帮助其继续尝试。

要求：
1) 仅给出步骤，不直接生成完整答案；
2) 每一步都应能在当前编辑器中立即尝试；
3) 最多给出 4 步；
4) 最后提醒学习者重新提交并观察哪个公开用例发生变化。
\end{quote}

\subsubsection*{请求数据格式（Input JSON）}

\begin{quote}\small\ttfamily
\{
\ \ "model":\ "deepseek-chat",\\
\ \ "messages":\ [\\
\ \ \ \ \{\\
\ \ \ \ \ \ "role":\ "system",\\
\ \ \ \ \ \ "content":\ "<上述系统提示词>"\\
\ \ \ \ \},\\
\ \ \ \ \{\\
\ \ \ \ \ \ "role":\ "user",\\
\ \ \ \ \ \ "content":\ \{\\
\ \ \ \ \ \ \ \ "exercise\_prompt":\ "请使用 HTML 和 CSS\\
\ \ \ \ \ \ \ \ \ \ 实现 3 列卡片布局...",\\
\ \ \ \ \ \ \ \ "source\_code":\ "学习者提交的完整\\
\ \ \ \ \ \ \ \ \ \ HTML/CSS 源码",\\
\ \ \ \ \ \ \ \ "error\_context":\ \{\\
\ \ \ \ \ \ \ \ \ \ "failing\_case":\ "mobile\_single\_column",\\
\ \ \ \ \ \ \ \ \ \ "actual\_columns":\ 2\\
\ \ \ \ \ \ \ \},\\
\ \ \ \ \ \ \ \ "attempt\_no":\ 5,\\
\ \ \ \ \ \ \ \ "hint\_level":\ 3,\\
\ \ \ \ \ \ \ \ "previous\_hints":\ [\\
\ \ \ \ \ \ \ \ \ \ "检查媒体查询断点条件",\\
\ \ \ \ \ \ \ \ \ \ "重写容器的列定义"\\
\ \ \ \ \ \ \ \ ]\\
\ \ \ \ \ \}\\
\ \ \ \ \}\\
\ \ ],\\
\ \ "temperature":\ 0.3,\\
\ \ "max\_tokens":\ 1000\\
\}
\end{quote}

\subsubsection*{返回数据格式（Output JSON）}

\begin{quote}\small\ttfamily
\{
\ \ "hint\_level":\ 3,\\
\ \ "summary":\ "请按以下步骤逐项检查并重新提交",\\
\ \ "action\_steps":\ [\\
\ \ \ \ \{\\
\ \ \ \ \ \ "step":\ 1,\\
\ \ \ \ \ \ "action":\ "将视口宽度手动设置为 375px",\\
\ \ \ \ \ \ "expected":\ "确认当前容器列数"\\
\ \ \ \ \},\\
\ \ \ \ \{\\
\ \ \ \ \ \ "step":\ 2,\\
\ \ \ \ \ \ "action":\ "调整媒体查询断点条件并测试",\\
\ \ \ \ \ \ "expected":\ "观察列数变化"\\
\ \ \ \ \},\\
\ \ \ \ \{\\
\ \ \ \ \ \ "step":\ 3,\\
\ \ \ \ \ \ "action":\ "在媒体查询内部重写容器的列定义",\\
\ \ \ \ \ \ "expected":\ "确认步骤 2 和 3 共同生效后列数为 1"\\
\ \ \ \ \},\\
\ \ \ \ \{\\
\ \ \ \ \ \ "step":\ 4,\\
\ \ \ \ \ \ "action":\ "重新提交代码，关注公开用例结果",\\
\ \ \ \ \ \ "expected":\ "公开用例应显示通过"\\
\ \ \ \ \}\\
\ \ ],\\
\ \ "final\_reminder":\ "提交后若仍有未通过的用例，请查看\\
\ \ \ \ 失败日志中的 actual\_columns 值来判断是否需继续调整"\\
\}
\end{quote}

三级提示最接近“教学支架”：\texttt{action\_steps} 提供了可逐项执行的操作列表，每项包含操作描述和预期观察结果；\texttt{final\_reminder} 鼓励学习者修改后重新提交。输入中增加了历史提示和尝试次数信息，使模型感知学习者已多次尝试但未成功，从而调整策略且仍不输出最终答案。

% ============ 小结 ============
综上，系统的 AI 辅助能力不是面向“自动写代码”，而是面向“在练习受阻时提供分层提示”。通过列出提示层级、系统提示词、输入输出 JSON 格式，能够使该设计更加具体、可验证。

\chapter{OpenAPI 接口完整列表}

本附录列出系统全部 95 个 API 端点，按访问权限分为公开接口、通用接口、学习者接口和管理端接口四个类别。所有路径均以 \path{/api/v1} 为前缀。

\subsection*{一、公开接口（无需认证）}

\begin{itemize}
  \item \textbf{POST} \path{/auth/register} \hfill 注册学习账号
  \item \textbf{POST} \path{/auth/learner/login} \hfill 学习者密码登录
  \item \textbf{POST} \path{/auth/admin/login} \hfill 管理端密码登录
  \item \textbf{POST} \path{/auth/refresh} \hfill 刷新 JWT 令牌
  \item \textbf{POST} \path{/auth/logout} \hfill 登出撤销会话
  \item \textbf{GET} \path{/health} \hfill 服务健康检查
\end{itemize}

\subsection*{二、通用接口（需 JWT 认证，不区分角色）}

\begin{itemize}
  \item \textbf{GET} \path{/me} \hfill 获取当前账号信息
\end{itemize}
