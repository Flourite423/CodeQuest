# Citation Policy

## Source Types
- Academic literature: for theory, concept definition, research status
- Project documents: for factual project scope and testing arrangement
- Source code: for implementation facts only
- Official documentation: for factual technology descriptions

## Rules
- Do not cite source file paths as academic literature.
- Do not use internal project docs to support broad educational theory claims.
- Use project docs to support requirement, architecture, and testing facts.
- Use official docs for framework facts.
- Use academic literature for gamification, mobile learning, and educational theory.

## Chapter Guidance
- Chapter 1: academic literature + project scope facts
- Chapter 2: academic literature + official technical references
- Chapter 3: project requirement documents
- Chapter 4: code and contract facts
- Chapter 5: code facts + testing plan
