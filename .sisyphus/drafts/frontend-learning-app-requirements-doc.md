# 前端学习APP需求分析文档

## 1. 项目概述

本项目旨在开发一款面向前端初学者的移动端学习APP，将HTML与CSS的学习过程设计为闯关冒险任务。用户通过完成每日挑战、答题闯关和实践编码任务来解锁关卡与奖励，同时整合必要的AI辅助功能和平台内社交互动机制，帮助学习者快速掌握前端基础技能。

## 2. 建设目标

1. **构建游戏化学习路径**：在MVP阶段上线10-15个线性闯关关卡，覆盖HTML与CSS核心知识点，帮助学习者系统掌握前端基础。
2. **提升学习完成率**：通过即时反馈与成就机制，帮助学习者保持学习动力，提升课程完成率。
3. **实现必要的AI学习辅助**：集成AI辅助能力（错误解释、必要提示），在5秒内响应用户求助，帮助学习者快速定位常见前端学习问题，降低学习中断率。
4. **支持移动端编码实践**：提供APP内代码编辑与实时预览功能，使学习者无需电脑即可完成基础编码练习。
5. **建立学习数据追踪体系**：记录用户学习时长、关卡进度、代码提交次数等核心指标，为管理员提供可视化数据报表。
6. **验证核心学习闭环**：在MVP阶段通过用户测试验证"学习-练习-闯关-反馈"闭环的有效性，收集用户反馈用于迭代优化。
7. **控制项目范围与成本**：严格按MVP范围交付，避免功能蔓延，确保毕业设计周期内可完成开发与论文撰写。
8. **形成可复用的产品方法论**：总结游戏化学习设计、AI辅助教学、移动端编码实践的经验，输出一套可指导后续迭代的轻量级需求与设计规范。

## 3. 目标用户

### 3.1 用户画像一：在校大学生"小明"

- **年龄范围**：19-22岁
- **职业背景**：计算机相关专业或非技术专业在读本科生
- **技术背景**：零基础或仅在学校课程中接触过少量HTML标签，对CSS完全陌生
- **学习动机**：希望通过自学掌握前端技能，为毕业设计或暑期实习做准备；对游戏化学习方式感兴趣，认为传统视频课程枯燥难以坚持
- **使用习惯**：每天使用APP 30-60分钟，主要在晚自习后或课间碎片时间学习；偏好闯关解锁的成就感，遇到难题会先尝试3次再求助AI；每周完成2-3个关卡即感到满意

### 3.2 用户画像二：职场新人"阿杰"

- **年龄范围**：23-26岁
- **职业背景**：刚入职互联网公司的产品、运营或设计岗位，希望转岗为前端开发工程师
- **技术背景**：了解互联网产品逻辑，但无实际编程经验；能看懂简单代码片段，无法独立编写完整页面
- **学习动机**：需要在3-6个月内达到能独立开发简单网页的水平，以便申请公司内部转岗或跳槽；学习时间有限，需要高效、结构化的学习路径
- **使用习惯**：工作日每天使用20-30分钟（通勤或午休时间），周末集中学习1-2小时；重视学习进度可视化，习惯在周末查看本周学习数据；对AI辅助功能依赖度较高，遇到报错会直接复制给AI分析

### 3.3 用户画像三：兴趣驱动自学者"莉莉"

- **年龄范围**：27-30岁
- **职业背景**：非互联网行业从业者（如行政、销售、教师等），出于个人兴趣或副业需求学习前端
- **技术背景**：完全零基础，对编程概念（变量、函数、循环）感到陌生；学习过程中容易因挫败感而放弃
- **学习动机**：希望掌握基础前端技能，能够独立搭建个人博客或简单静态网站；不追求成为专业工程师，但希望获得"我会编程"的成就感
- **使用习惯**：学习时间不规律，可能连续几天高频使用，也可能因忙碌而中断1-2周；对游戏化激励机制敏感，成就徽章和连续学习天数能有效促使其回归；偏好图文教程而非视频，因为可以按自己的节奏阅读

## 4. 使用场景

### 4.1 场景一：日常碎片化学习

- **触发条件**：用户在地铁通勤或午休时打开APP，看到首页展示的"今日任务"
- **用户操作**：点击进入每日挑战，完成当日系统生成的一道限时练习题
- **系统响应**：即时显示答题结果与解析；完成挑战后弹出"今日挑战完成"提示，累计连续学习天数+1
- **预期结果**：用户在数分钟内完成学习，获得即时成就感，强化每日学习习惯

### 4.2 场景二：闯关挑战与编码实践

- **触发条件**：用户周末有1小时空闲时间，想集中学习新内容，在课程页面看到"CSS布局基础"章节已解锁
- **用户操作**：进入关卡地图选择第5关"Flexbox布局挑战"，观看图文教程后进入编码练习环节，在代码编辑器中编写Flexbox相关CSS代码
- **系统响应**：代码编辑器支持实时预览，用户每修改一行代码，预览即时更新页面效果；点击提交后，系统自动运行测试用例，检查是否满足关卡要求（如"实现水平居中布局"）；根据代码质量和提交次数给出1-3星评分
- **预期结果**：用户在45分钟内完成学习+练习+闯关全流程，掌握Flexbox核心概念，获得3星通关奖励并解锁下一关卡

### 4.3 场景三：AI求助与错误诊断

- **触发条件**：用户在编码练习中尝试实现"响应式导航栏"，代码提交后测试用例连续3次未通过，系统检测到用户在同一关卡停留超过10分钟
- **用户操作**：点击右下角的AI助手图标，选择"帮我分析错误"选项，系统自动将当前代码和错误信息发送给AI
- **系统响应**：AI在3-5秒内返回错误分析结果，指出"未设置@media查询的断点"和"导航栏在小屏幕下未隐藏"两个问题，并提供修复建议代码片段；用户可一键复制建议代码到编辑器，或查看系统返回的"媒体查询"知识点解释卡片
- **预期结果**：用户在2分钟内定位并修复问题，成功通过关卡；同时理解了媒体查询的概念，实现"从错误中学习"

### 4.4 场景四：社交互动与学习动态

- **触发条件**：用户在社交页面看到"好友小华正在挑战第8关"的动态
- **用户操作**：浏览好友的学习动态，查看好友的通关成绩和获得的徽章，添加好友，查看排行榜，分享学习成果
- **系统响应**：系统展示好友最近的学习动态列表，包括通关关卡、获得徽章、连续学习天数等信息；支持在平台内展示系统生成的学习成果卡片
- **预期结果**：用户通过社交互动获得学习动力，形成良性竞争氛围；组队挑战功能在后续迭代中实现

### 4.5 场景五：进度查看与学习激励

- **触发条件**：用户连续学习7天后，在"我的"页面点击"学习数据"入口
- **用户操作**：查看本周学习时长统计（共3.5小时）、完成的关卡数量（5关）、代码提交次数（42次）、连续学习天数（7天）等数据
- **系统响应**：系统生成可视化学习报告，包括学习时长趋势图、知识点掌握雷达图、与上周的对比数据；同时显示已获得的成就徽章（"连续7天学习"、"HTML新手"、"代码达人"），并提示"再完成3关即可解锁CSS大师徽章"
- **预期结果**：用户清晰感知到自己的进步，获得正向反馈；目标导向的徽章机制激励用户制定下周学习计划，提升留存率

### 4.6 场景六：管理后台内容运营

- **触发条件**：管理员登录Web管理后台，发现"HTML基础"课程的用户完成率仅为45%，低于预期目标
- **用户操作**：在数据统计页面查看该课程的详细数据，发现第3关"表单元素"的放弃率偏高；进入内容管理模块，查看该关卡的题目难度和用户反馈
- **系统响应**：系统展示该关卡的详细数据，显示大部分用户在编码练习环节停留时间过长；管理员根据数据判断该关卡难度过高，调整题目要求并增加AI提示触发阈值
- **预期结果**：管理员完成内容优化后，该关卡的完成率应有提升；数据驱动的运营方式确保课程质量持续优化

## 5. 系统范围

- 学习者端：移动端APP（Android/iOS）
- 管理后台：Web端管理界面
- 包含模块：课程学习、编码练习、闯关任务、AI辅助、社交互动、用户管理、内容管理
- 不包含：PC端学习平台、第三方LMS集成、实时音视频功能

## 6. 角色定义

本系统仅设两类角色，角色与终端的对应关系如下：

- **学习者**：面向前端初学者的普通用户，通过**移动端 APP**（Android/iOS）完成学习、练习、闯关和社交互动。学习者仅操作自身数据，无法进入管理后台。
- **管理员**：负责内容运营与系统管理的统一角色，通过**Web 管理后台**进行内容发布、用户管理、数据查看和系统配置。后台为统一角色，不拆分教师、运营或管理员等多套独立系统。

## 8. 信息架构

### 8.1 学习者端页面结构

- 首页：学习进度概览、今日任务、学习入口导航
- 课程：课程列表、课程详情、章节学习、图文内容
- 练习：编码练习入口、代码编辑器、运行结果、AI提示
- 闯关：关卡地图、关卡详情、挑战任务、奖励结算
- 社交：好友列表、好友学习行为动态、成果分享、排行榜
- 我的：个人资料、学习记录、成就徽章、设置

### 8.2 核心业务流程

#### 8.2.1 学习者学习主流程
- 注册或登录账户 → 浏览课程目录 → 进入章节学习 → 点击“完成学习” → 解锁练习与测验 → 完成闯关任务 → 获取奖励与进度更新。

#### 8.2.2 每日挑战流程
- 用户进入首页或每日挑战页面 → 查看当日挑战 → 完成作答或代码提交 → 系统判定结果 → 发放经验值与连续学习奖励。

#### 8.2.3 AI辅助流程
- 用户在练习失败或主动提问时唤起 AI → 系统给出错误解释或学习提示 → 用户继续练习、复习知识点或返回课程内容。

#### 8.2.4 管理后台主流程
- 管理员登录后台 → 配置课程、题目与关卡 → 发布学习内容与学习公告 → 处理用户反馈与基础审核事项 → 查看用户学习数据 → 调整内容与基础规则。

## 9. 学习内容结构

### 9.1 课程单元

**定义**
课程单元是围绕单一技术域组织的完整学习模块，属于学习内容的第一级聚合单位。整体课程体系面向 HTML、CSS、JavaScript 三大前端基础技术域设计；其中 MVP 阶段优先上线 2 个课程单元，分别为 HTML 基础与 CSS 基础，JavaScript 基础单元列入后续迭代。

**输入**
- 学习者账号已完成注册并登录
- 学习者已解锁该课程单元（首个单元默认解锁，后续单元需完成前置单元）
- 课程单元配置数据（单元名称、描述、包含章节列表、单元图标）

**输出**
- 学习者学习进度记录（单元完成百分比、已解锁章节列表）
- 单元完成状态（未开始/进行中/已完成）
- 单元完成奖励（经验值、解锁下一课程单元）

**触发条件**
- 学习者首次进入课程列表页时，首个课程单元自动解锁
- 学习者完成上一课程单元的全部章节后，系统自动解锁下一课程单元

**完成规则**
- 学习者完成课程单元内全部章节的学习标记，且通过单元结业测验，视为单元完成
- 单元结业测验需达到60分及以上方可通过
- 单元完成后，学习者获得该单元对应经验值，并解锁下一课程单元

### 9.2 章节

**定义**
章节是课程单元内的二级学习单位，围绕一个具体知识点或技能点展开，包含概念讲解、代码示例、交互演示等教学内容。每个课程单元包含3-5个章节。

**输入**
- 学习者已解锁所属课程单元
- 学习者已完成该章节的所有前置章节
- 章节配置数据（章节标题、教学内容、预计学习时长、关联练习ID列表）

**输出**
- 章节学习进度（已读内容百分比、学习时长）
- 章节完成标记
- 解锁关联练习与下一章节

**触发条件**
- 学习者点击进入已解锁的章节详情页
- 学习者完成上一章节的学习标记后，下一章节自动解锁

**完成规则**
- 学习者阅读完章节全部教学内容（滑动至底部或停留时长达到章节预计时长的大部分），并点击"完成学习"按钮，视为章节完成
- 章节完成后，自动解锁该章节关联的练习题和下一章节
- 若章节为单元内最后一个章节，则解锁单元结业测验

### 9.3 练习

**定义**
练习是与章节知识点一一对应的编码任务，要求学习者在APP内编写代码并提交，系统自动评测代码正确性。练习分为概念选择题和编码实操题两类。

**输入**
- 学习者已完成关联章节的学习
- 练习配置数据（题目描述、初始代码模板、测试用例、参考答案、分值）
- 学习者提交的代码或选择的答案

**输出**
- 评测结果（通过/失败、得分、错误提示）
- 练习完成标记
- 学习记录更新（练习次数、最佳得分）

**触发条件**
- 学习者完成关联章节后，练习入口自动解锁
- 学习者从章节详情页或练习列表页进入练习

**完成规则**
- 概念选择题：选择正确答案并提交，答对即视为完成，得满分
- 编码实操题：代码通过全部测试用例即视为完成，获得该练习对应分值
- 练习允许重复提交，系统记录最佳成绩
- 单次练习不通过不影响章节完成状态，但影响学习者的知识掌握度评估

### 9.4 测验

**定义**
测验是对学习者阶段性知识掌握情况的评估工具，分为章节小测和单元结业测验两种。章节小测位于章节末尾，单元结业测验位于课程单元末尾。

**输入**
- 学习者已完成测验关联的全部章节（章节小测）或全部单元章节（单元结业测验）
- 测验配置数据（题目列表、选项、正确答案、分值、限时）

**输出**
- 测验得分（百分制）
- 测验结果（通过/失败）
- 错题解析与知识点回顾建议

**触发条件**
- 章节小测：学习者完成该章节全部教学内容后自动触发
- 单元结业测验：学习者完成单元内全部章节后解锁

**完成规则**
- 章节小测：得分达到60分及以上视为通过，并记录该章节的知识掌握结果；未通过可重新答题
- 单元结业测验：得分达到60分及以上视为通过，完成该课程单元；未通过需24小时后方可重测
- 测验限时完成，超时自动提交已答题目

### 9.5 每日挑战

**定义**
每日挑战是系统每日刷新的一道限时练习题，内容从已解锁课程单元的知识点中随机抽取，用于巩固已学内容并激励持续学习。

**输入**
- 学习者当日首次打开APP或进入每日挑战页面
- 学习者已解锁至少一个课程单元
- 每日挑战配置数据（题目内容、测试用例、限时、奖励规则）

**输出**
- 挑战结果（成功/失败、用时、得分）
- 当日挑战完成标记
- 奖励发放（经验值、连续学习天数+1）

**触发条件**
- 每日0点系统刷新当日挑战内容
- 学习者主动进入每日挑战页面
- 学习者完成当日挑战后，当日不可再次参与

**完成规则**
- 学习者在限时内完成挑战并通过评测，视为挑战成功
- 挑战成功获得当日经验值奖励，连续学习天数+1
- 挑战失败不扣除任何资源，当日不可重试，次日刷新后可再次参与
- 连续完成每日挑战7天可获得额外经验值奖励与"坚持不懈"徽章

### 9.6 闯关任务

**定义**
闯关任务是将多个章节的学习、练习和测验整合为线性关卡的游戏化学习节点。MVP阶段设置10-15个闯关关卡，每个关卡对应一个或多个章节的核心知识点。

**输入**
- 学习者已解锁该关卡（首个关卡默认解锁）
- 关卡配置数据（关卡名称、关卡目标、包含任务列表、星级评定标准、奖励配置）
- 学习者完成关卡内各项任务的提交数据

**输出**
- 关卡完成状态（未解锁/已解锁/进行中/已完成）
- 星级评定（1-3星）
- 通关奖励（经验值、成就徽章、解锁下一关）

**触发条件**
- 学习者完成上一关卡后，下一关卡自动解锁
- 学习者从闯关地图页点击进入已解锁关卡

**完成规则**
- 学习者完成关卡内全部必做任务（包括学习指定章节、完成编码练习、通过关卡测验），视为通关
- 星级评定标准：
  - 1星：完成全部必做任务，但测验得分低于80分或练习提交次数超过3次
  - 2星：完成全部必做任务，测验得分80分及以上且练习提交次数不超过3次
  - 3星：完成全部必做任务，测验得分90分及以上且首次提交即通过全部练习
- 通关后解锁下一关卡，未通关可重复挑战
- 关卡内任务无时间限制，但鼓励学习者一次性完成

### 9.7 任务模型

**定义**
任务模型是描述学习者在系统中可执行的学习活动及其相互关系的概念框架。MVP阶段包含以下任务类型：课程单元学习、章节阅读、编码练习、章节小测、单元结业测验、每日挑战、闯关任务。各类任务通过解锁条件形成有向依赖关系，构成完整的学习路径。

**任务类型**
- 课程单元：围绕单一技术域的完整学习模块
- 章节：课程单元内的知识点单元，包含图文教学内容
- 练习：与章节对应的编码或选择题目，用于巩固知识
- 测验：章节小测与单元结业测验，用于评估掌握情况
- 每日挑战：每日刷新的限时练习题，用于持续学习激励
- 闯关任务：整合学习、练习与测验的游戏化关卡

### 9.8 对象间前后置关系与解锁条件

**整体解锁链**
课程单元 -> 章节 -> 练习/章节小测 -> 单元结业测验 -> 下一课程单元
闯关任务与上述学习路径并行，关卡解锁依赖于前置关卡的完成。

**详细解锁规则**

| 对象 | 解锁条件 | 前置依赖 |
|------|---------|---------|
| 首个课程单元（HTML基础） | 学习者注册并登录后自动解锁 | 无 |
| 后续课程单元 | 完成上一课程单元的全部章节并通过单元结业测验 | 上一课程单元完成 |
| 首个章节 | 所属课程单元解锁后自动解锁 | 课程单元解锁 |
| 后续章节 | 完成上一章节的学习标记 | 上一章节完成 |
| 练习 | 完成关联章节的学习标记后自动解锁 | 关联章节完成 |
| 章节小测 | 完成该章节全部教学内容后解锁 | 该章节完成 |
| 单元结业测验 | 完成单元内全部章节后解锁 | 单元内全部章节完成 |
| 每日挑战 | 学习者已解锁至少一个课程单元，且当日未参与过 | 至少一个课程单元解锁 |
| 首个闯关关卡 | 学习者注册并登录后自动解锁 | 无 |
| 后续闯关关卡 | 完成上一关卡并获得至少1星评价 | 上一关卡通关 |

**评分规则汇总**

| 对象 | 通过标准 | 失败标准 | 星级/等级评定 |
|------|---------|---------|-------------|
| 章节 | 完成学习标记 | 未完成学习标记 | 无 |
| 练习（选择） | 答对题目 | 答错题目 | 无 |
| 练习（编码） | 通过全部测试用例 | 未通过全部测试用例 | 无 |
| 章节小测 | 得分 >= 60分 | 得分 < 60分 | 无 |
| 单元结业测验 | 得分 >= 60分 | 得分 < 60分 | 无 |
| 每日挑战 | 限时内通过评测 | 超时或评测未通过 | 无 |
| 闯关任务 | 完成全部必做任务 | 未完成全部必做任务 | 1-3星（见9.6节） |

**最小完成路径**
学习者完成学习的最小路径为：解锁课程单元 -> 依次完成各章节学习标记并完成关联练习 -> 参加章节小测记录掌握情况 -> 通过单元结业测验 -> 完成课程单元。下一章节的解锁仅依赖前置章节学习完成；章节小测用于记录掌握情况，不单独阻塞下一章节解锁。闯关任务作为游戏化增强路径与课程学习并行，学习者可通过完成闯关关卡获得额外激励，但不影响课程单元的正常解锁流程。

## 10. 学习者端功能需求

### 10.1 账户与档案

**功能说明**
学习者通过手机号+验证码完成注册与登录，建立个人学习档案。注册后系统自动生成默认昵称与头像，学习者可在个人中心编辑昵称、上传头像，并设置学习偏好（每日目标时长、难度偏好、学习计划提醒时间）。账户信息用于关联学习进度、成就记录与社交关系。

**输入**
- 手机号（11位中国大陆号码）
- 短信验证码（6位数字）
- 昵称（2-20个字符，支持中英文、数字）
- 头像图片（JPG/PNG格式，不超过2MB）
- 学习偏好：每日目标时长（15/30/60分钟）、难度偏好（入门/标准/挑战）、学习计划提醒时间（HH:MM，应用内展示，不涉及主动推送）

**输出**
- 注册/登录成功状态与认证令牌
- 个人资料页展示（昵称、头像、注册时间、学习天数）
- 学习偏好保存结果
- 账户安全状态（登录设备列表、最近登录时间）

**业务规则**
- 同一手机号24小时内最多发送5条验证码
- 验证码有效期为5分钟，过期需重新获取
- 昵称不可包含敏感词，系统调用敏感词库校验
- 头像上传后生成适配展示的缩略图版本
- 学习偏好可随时修改，次日生效

**优先级**：P0（必须有）

**异常处理**
- 网络中断：提示"网络连接失败，请检查网络后重试"，保留已填写的手机号
- 输入验证失败：手机号格式错误时高亮输入框并提示"请输入正确的手机号"；验证码错误时提示"验证码错误，请重新输入"
- 重复提交：点击发送验证码后按钮进入60秒倒计时，防止重复点击
- 权限不足：未登录用户访问个人中心时自动跳转登录页

---

### 10.2 课程学习

**功能说明**
学习者浏览课程目录，查看已解锁与未解锁的课程单元及章节。进入章节后阅读图文教学内容（MVP阶段以图文为主，暂不含视频），系统记录学习进度。完成章节阅读后标记为"已完成"，自动解锁关联练习与下一章节。

**输入**
- 学习者点击课程目录或章节列表
- 章节内容滑动/阅读行为
- "完成学习"按钮点击

**输出**
- 课程目录树（含解锁状态、完成百分比）
- 章节图文内容
- 学习进度记录（已读百分比、学习时长）
- 章节完成标记与解锁提示

**业务规则**
- 首个课程单元（HTML基础）注册后自动解锁，CSS基础单元在HTML基础完成后解锁
- 章节解锁依赖前置章节完成
- 章节完成条件：学习者完成章节阅读后点击“完成学习”按钮
- 课程单元完成需通过单元结业测验（得分>=60分）
- 学习进度应及时保存并在重新进入时正确恢复

**优先级**：P0（必须有）

**异常处理**
- 网络中断：已加载的章节内容可离线阅读，进度在恢复网络后批量同步
- 内容加载失败：显示"内容加载失败"并提供"重试"按钮
- 重复提交：同一章节"完成学习"按钮点击后进入loading状态，防止重复标记
- 权限不足：未解锁章节显示锁图标，点击时提示"请先完成前置章节"

### 10.3 编码练习

**功能说明**
学习者在APP内置代码编辑器中编写HTML/CSS代码，编辑器支持语法高亮与自动补全。编写完成后可实时预览代码运行效果，点击提交后系统自动评测代码正确性并返回通过/失败结果与错误提示。

**输入**
- 学习者在代码编辑器中输入的代码文本
- 提交按钮点击事件
- 运行/预览按钮点击事件

**输出**
- 代码编辑器界面（含行号、语法高亮）
- 实时预览渲染结果（内置预览组件）
- 评测结果（通过/失败、得分、错误信息、测试用例详情）
- 练习完成标记与最佳成绩记录

**业务规则**
- 编码练习解锁条件：完成关联章节的学习标记
- 代码运行限制执行时间5秒
- 评测标准：代码通过全部测试用例视为通过
- 允许重复提交，系统记录最佳成绩
- 代码在本地自动保存草稿，防止意外退出丢失

**优先级**：P0（必须有）

**异常处理**
- 网络中断：代码草稿保存在本地，提交时提示"网络异常，请稍后重试"
- 输入验证失败：空代码提交时提示"请输入代码后再提交"
- 重复提交：提交后按钮置为loading状态，评测完成前不可再次提交
- 代码运行超时：提示"代码运行超时，请检查是否存在死循环"
- 权限不足：未解锁练习显示锁图标，点击提示"请先完成关联章节"

---

### 10.4 测验闯关

**功能说明**
学习者进入闯关地图查看各关卡解锁状态，选择已解锁关卡进行挑战。每个关卡整合章节学习、编码练习与关卡测验，学习者需完成全部必做任务方可通关。系统根据测验得分与练习提交次数评定1-3星，通关后发放奖励并解锁下一关。

**输入**
- 学习者点击关卡地图中的已解锁关卡
- 关卡内各任务（学习、练习、测验）的完成数据
- 测验答案提交

**输出**
- 关卡地图（含解锁状态、星级评定、进度百分比）
- 关卡详情页（任务列表、奖励预览）
- 通关结果页（星级、经验值、解锁提示）
- 本关通关成绩展示（按星级与用时排序，作为全局排行榜的数据来源之一）

**业务规则**
- MVP阶段设置10-15个线性闯关关卡
- 首个关卡注册后自动解锁，后续关卡需前置关卡通关（>=1星）
- 星级评定：
  - 1星：完成全部必做任务，但测验得分<80分或练习提交>3次
  - 2星：完成全部必做任务，测验得分>=80分且练习提交<=3次
  - 3星：完成全部必做任务，测验得分>=90分且首次提交通过全部练习
- 未通关可重复挑战，已通关关卡可回顾但不再获得奖励

**优先级**：P0（必须有）

**异常处理**
- 网络中断：关卡地图使用本地缓存，关卡进度在恢复网络后同步
- 输入验证失败：测验答案未填写完整时提示"请完成所有题目"
- 重复提交：测验提交后进入评分状态，防止重复提交
- 权限不足：未解锁关卡点击时提示"请先通关前置关卡"

### 10.5 每日挑战

**功能说明**
系统每日0点从学习者已解锁课程单元的知识点中随机抽取一道限时练习题，并在首页与每日挑战页面展示。学习者在当日首次打开APP或主动进入每日挑战页面即可参与。限时内完成并通过评测视为挑战成功，获得经验值奖励并累计连续学习天数。

**输入**
- 学习者当日首次打开APP或进入每日挑战页面
- 学习者提交的答案或代码
- 系统每日0点刷新的挑战配置

**输出**
- 每日挑战题目（含题目描述、限时倒计时）
- 挑战结果（成功/失败、用时、得分）
- 奖励结果提示（经验值、连续天数）
- 连续挑战7天额外奖励提示

**业务规则**
- 每日挑战内容从已解锁课程单元的知识点中随机抽取
- 每日0点刷新，学习者当日仅可参与一次
- 挑战限时5分钟，超时自动提交
- 挑战成功：获得当日经验值奖励，连续学习天数+1
- 挑战失败：不扣除任何资源，当日不可重试
- 连续完成每日挑战7天可获得额外经验值奖励与"坚持不懈"徽章

**优先级**：P1（重要）

**异常处理**
- 网络中断：挑战页面加载失败时提示"网络异常，请检查网络后重试"
- 输入验证失败：未作答提交时提示"请先完成题目"
- 重复提交：挑战提交后锁定，当日不可再次参与
- 权限不足：未解锁任何课程单元的用户提示"请先开始学习课程"

### 10.6 奖励成长

**功能说明**
学习者通过完成课程、通关关卡、连续挑战等行为获得经验值与成就徽章。经验值累计提升学习者等级，等级提升解锁新称号与头像框。成就徽章分为学习类、挑战类、社交类，展示在个人档案中。系统记录连续学习天数，断签后连续天数归零。

**输入**
- 学习者完成课程、关卡、挑战等行为事件
- 系统计算的奖励规则

**输出**
- 经验值变动提示（+XX经验值）
- 等级提升动画与称号变更
- 成就徽章解锁弹窗
- 个人档案中的徽章墙与等级展示
- 连续学习天数状态（连续天数、今日是否已完成）

**业务规则**
- 经验值来源：完成章节（+10）、通过练习（+20）、通关关卡（+50-150，按星级）、完成每日挑战（+30）、连续7天完成每日挑战（+100）
- 等级计算：每累计一定经验值提升1级
- 成就徽章示例："HTML新手"（完成HTML基础单元）、"代码达人"（累计提交代码50次）、"坚持不懈"（连续7天完成每日挑战）
- 连续学习天数规则：每日完成至少一个学习行为（课程/练习/挑战）即算打卡，断签后连续天数归零
- MVP阶段徽章数量控制在10个以内

**优先级**：P1（重要）

**异常处理**
- 网络中断：奖励结果在网络恢复后补记并展示，避免重复发放
- 重复提交：同一行为仅触发一次奖励
- 权限不足：无

### 10.7 AI提示

**功能说明**
学习者在学习过程中可在必要时唤起AI助手。AI仅支持错误解释与必要提示：当学习者提交代码失败、长时间卡住或主动求助时，系统基于当前代码、错误信息和练习上下文，给出问题原因说明、排查方向与简短提示。当学习者在同一关卡停留超过10分钟或连续3次提交失败时，系统展示AI提示入口。

**输入**
- 学习者输入的求助问题
- 学习者当前代码与错误信息（用于错误分析场景）
- 系统触发的提示请求（停留超时、多次失败）

**输出**
- AI回答文本（错误分析、排查建议、简短提示）
- 相关知识点提示卡片
- AI提示入口展示状态

**业务规则**
- AI响应时间目标：5秒内返回结果
- AI仅支持错误解释与必要提示两类能力
- AI仅围绕当前课程、练习与报错场景提供单次辅助，不支持扩展对话
- AI不直接修改用户代码，不提供代码补全或整段答案生成
- AI不替代人工学习，关键知识点需学习者自主理解
- 错误解释场景：系统自动将当前代码与测试用例错误信息发送至AI，AI返回原因分析与修复建议
- 必要提示场景：AI基于当前错误或当前练习步骤提供知识点提示卡片与排查方向

**优先级**：P1（重要）

**异常处理**
- 网络中断：AI请求失败时提示"AI助手暂时离线，请检查网络后重试"
- 输入验证失败：空问题提交时提示"请输入您的问题"
- 重复提交：AI回答生成中锁定输入框，防止重复发送
- AI服务不可用：提示"AI助手暂时离线，您可以稍后重试或查看文档"

---

### 10.8 社交互动

**功能说明**
学习者可通过搜索用户名或分享邀请码添加好友，查看好友的学习行为动态（如"小华通关了第5关"）。支持生成学习成果卡片并在平台内展示或保存至本地。系统维护关卡通关速度榜、周积分榜、连续学习天数榜三类排行榜。MVP阶段仅支持平台内的基础好友关系、好友学习行为动态查看、成果展示和排行榜，不涉及第三方平台互动、开放社区或UGC动态广场，组队挑战功能列入后续迭代。

**输入**
- 搜索关键词或邀请码
- 分享卡片类型选择（通关成绩/徽章/连续天数）

**输出**
- 好友列表（含最近动态）
- 好友学习行为动态流（好友通关、获得徽章等最近学习行为）
- 成果卡片（系统生成，支持平台内展示与保存本地）
- 排行榜（关卡通关速度榜、周积分榜、连续学习天数榜，前50名）

**业务规则**
- 好友添加上限：每人最多50个好友
- 好友学习行为动态仅展示最近7天的行为
- 排行榜每日0点刷新，基于前一日数据计算排名
- MVP阶段社交功能以基础互动为主，不发展为通用社交平台

**优先级**：P1（重要）

**异常处理**
- 网络中断：好友列表使用本地缓存，动态在恢复网络后刷新
- 输入验证失败：搜索空关键词时提示"请输入用户名或邀请码"
- 权限不足：非好友用户无法查看详细学习数据

### 10.9 学习进度与个人中心

**功能说明**
学习者在"我的"页面查看个人学习数据总览，包括学习时长统计、关卡进度、代码提交次数、连续学习天数、知识点掌握雷达图等。数据以可视化图表形式呈现，支持按周/月维度切换。同时可查看已获得的成就徽章、编辑个人资料、管理学习偏好与账号安全设置。

**输入**
- 学习者进入"我的"页面
- 时间维度切换（本周/本月/全部）
- 个人资料编辑提交

**输出**
- 学习数据总览（学习时长、完成关卡数、代码提交次数、连续天数）
- 可视化图表（学习时长趋势图、知识点掌握雷达图、与上周对比）
- 成就徽章墙（已解锁/未解锁状态）
- 个人资料编辑表单
- 设置页面（学习偏好、账号安全）

**业务规则**
- 学习时长统计：累计在APP内活跃且处于学习相关页面的时间
- 数据刷新频率：进入页面时实时拉取，页面内切换维度使用本地数据
- 知识点掌握度：基于章节完成率、练习正确率、测验得分综合计算
- 连续学习天数：每日完成至少一个学习行为即算打卡
- 数据保留期限：学习记录永久保留，好友学习行为动态仅保留最近7天

**优先级**：P1（重要）

**异常处理**
- 网络中断：已加载的数据可离线查看，图表使用上次缓存数据
- 输入验证失败：昵称编辑不符合规则时提示具体错误
- 重复提交：资料保存按钮点击后进入loading状态
- 权限不足：未登录用户访问"我的"页面时自动跳转登录页

## 11. 管理后台功能需求

**定位说明**
- 管理后台为 **Web 端管理界面**，面向管理员角色，通过浏览器访问。
- 后台仅承担内容运营、用户管理、数据查看和系统配置职责，**不承担学员学习主流程**。
- 后台采用**统一角色**，不拆分教师后台、运营后台、管理员后台等多套独立系统，所有管理权限集中在单一后台中。

### 11.1 课程与章节管理

**功能说明**
提供课程单元的创建、编辑、上下架和删除功能，支持章节内容的组织与排序。管理员可配置课程基本信息、章节结构和教学素材，控制课程对学员的可见状态。

**输入**
- 课程基本信息（课程名称、描述、封面图、难度等级、预计学习时长）
- 章节内容（章节标题、图文教学内容、预计学习时长）
- 章节排序与关联关系（前置章节、章节顺序）
- 课程状态（草稿/已发布/已下架）

**输出**
- 课程列表（含发布状态、创建时间、学员完成率概览）
- 章节结构树（层级关系、解锁条件）
- 课程发布/下架操作结果
- 课程编辑历史记录

**业务规则**
- 课程发布前，必须包含至少1个章节且章节内容不为空
- 已发布课程被下架后，新学员无法选课，但已选学员可继续学习
- 章节顺序调整时，系统自动更新关联的解锁逻辑
- 删除课程前，系统提示该课程已有学员学习记录，确认后方可删除

**优先级：P0**

### 11.2 题目与挑战管理

**功能说明**
管理练习题、编码实操题和闯关关卡的配置。支持题目创建、编辑、测试用例配置、难度标注和关卡奖励规则设定。

**输入**
- 题目基本信息（题目标题、题目描述、题目类型：选择/编码）
- 选择题配置（选项列表、正确答案、解析）
- 编码题配置（初始代码模板、测试用例、参考答案、运行时限）
- 闯关关卡配置（关卡名称、关联章节、任务列表、星级评定标准、奖励规则）
- 题目难度标签（简单/中等/困难）

**输出**
- 题目列表（含类型、难度、使用次数、正确率概览）
- 测试用例执行结果预览
- 关卡结构地图（关卡顺序、解锁条件、奖励配置）
- 题目上下架状态

**业务规则**
- 编码题必须配置至少1个测试用例，且参考答案需通过全部测试用例验证
- 闯关关卡关联的章节必须是已发布状态
- 题目被学员作答后，编辑题目不影响历史作答记录
- 关卡奖励配置需满足：经验值 >= 0，徽章需在系统预定义列表中选择

**优先级：P0**

### 11.3 用户与学习记录管理

**功能说明**
查看和管理注册用户的基本信息、学习进度、活跃度和账号状态。支持用户搜索、筛选、详情查看和账号禁用/启用操作。

**输入**
- 用户查询条件（用户ID、昵称、手机号、注册时间范围、账号状态）
- 账号状态变更（禁用/启用）
- 学习记录筛选（课程、时间范围、完成状态）

**输出**
- 用户列表（用户ID、昵称、注册时间、最近活跃时间、账号状态）
- 用户详情（个人资料、学习进度、关卡完成情况、成就徽章）
- 用户学习记录（学习时长、章节完成数、练习提交次数、代码运行次数）
- 账号状态变更结果

**业务规则**
- 管理员可查看全部用户数据，但不可修改用户个人资料（除账号状态外）
- 禁用账号后，该用户无法登录APP，但历史数据保留
- 用户列表支持按注册时间、活跃度排序和分页
- 学习记录数据延迟不超过1小时

**优先级：P0**

### 11.4 内容审核与反馈处理

**功能说明**
提供轻量级内容审核与反馈处理能力，用于管理用户昵称、头像以及题目纠错反馈，保证基础内容质量与平台秩序。成果分享卡片为系统生成，不涉及用户自定义内容审核。

**输入**
- 用户反馈内容（题目纠错、内容举报）
- 待处理资料（昵称、头像）
- 处理动作（通过、驳回、下架）

**输出**
- 待处理反馈列表
- 审核处理结果
- 处理记录与状态更新结果

**业务规则**
- MVP阶段仅处理用户昵称/头像的基础资料审核、题目纠错与内容反馈处理
- 成果分享卡片由系统根据学习数据自动生成，不支持用户自定义文本输入，无需额外内容审核
- 处理结果需对用户可见
- 审核与反馈处理保留基础记录，便于后续追踪

**优先级：P1**

### 11.5 公告与基础运营配置

**功能说明**
支持发布学习公告、维护每日挑战基础配置与展示类运营文案，满足 MVP 阶段最小运营需求。

**输入**
- 公告内容（标题、正文、有效期）
- 每日挑战基础配置
- 展示类运营文案

**输出**
- 公告列表与发布状态
- 每日挑战配置结果
- 运营文案更新结果

**业务规则**
- 公告仅用于学习相关说明与展示信息
- 每日挑战配置修改后于下一刷新周期生效
- 运营配置保持轻量，不扩展为复杂活动系统

**优先级：P1**

### 11.6 数据统计看板

**功能说明**
提供全局数据可视化看板，展示课程学习情况、用户活跃度、题目正确率、AI使用频次等核心指标，支持数据筛选和报表导出。

**输入**
- 时间范围筛选（今日/近7天/近30天/自定义）
- 课程筛选（全部/指定课程）
- 数据维度选择（用户/课程/题目/AI）

**输出**
- 核心指标卡片（注册用户数、日活跃用户、课程平均完成率、人均学习时长）
- 课程数据（各课程完成率、各章节放弃率）
- 题目数据（各题目正确率分布、难度与正确率对比、高频错误分析）
- 用户数据（新增用户趋势、留存率曲线、活跃时段分布、连续学习天数分布）
- AI数据（AI辅助功能使用频次、平均响应时间）
- 数据报表导出（CSV格式）

**业务规则**
- 看板数据每日凌晨3点自动更新，实时数据延迟不超过1小时
- 课程完成率 = 完成该课程全部章节的学员数 / 该课程总学员数 * 100%
- 留存率按日留存和7日留存分别计算
- 报表导出最大支持90天数据范围

**优先级：P1**

### 11.7 系统基础配置

**功能说明**
配置系统级基础规则，包括AI辅助规则、积分规则、成就徽章定义与操作记录查看等基础设置。

**输入**
- AI配置（辅助规则、每日使用次数上限）
- 积分规则（章节完成奖励、练习通过奖励、连续学习奖励、闯关星级奖励）
- 徽章配置（徽章名称、解锁条件、图标）
- 系统参数（基础展示文案、规则开关）

**输出**
- 配置项列表（配置名称、当前值、最后修改时间）
- AI辅助规则预览
- 积分规则明细表
- 徽章列表与解锁统计
- 系统操作记录（操作人、操作类型、操作时间）

**业务规则**
- 系统配置修改后应在合理时间内生效
- 积分规则修改不影响已发放的历史积分
- 徽章解锁条件修改后，已解锁用户保留徽章，新用户按新条件执行
- 系统操作记录保留90天，超期自动清理
- 管理员密码修改和账号禁用操作需二次确认

**优先级：P0（支撑后台运营的基础配置能力，与其他后台模块同步实现）**

## 12. AI能力边界

### 12.1 MVP阶段AI能力范围

MVP阶段AI功能定位为"辅助工具"而非"教学代理"，所有建议仅供参考，不替代人工审核与学习者自主思考。

#### 12.1.1 实时代码检查

**功能说明**
MVP阶段不提供独立的实时代码检查能力，避免AI功能超出必要范围。代码正确性仍由既有评测与错误反馈机制承担。

**能力范围**
- 本阶段不作为独立AI能力开放
- 代码评测结果由系统基础评测机制返回
- AI仅在用户失败后参与解释与提示

#### 12.1.2 错误解释

**功能说明**
当用户代码未通过测试用例或出现运行错误时，系统解析错误信息并用通俗易懂的语言向学习者解释问题原因和修复方向。

**能力范围**
- 错误原因说明（用中文描述）
- 修复建议（1-3条具体可行的修改方向）
- 相关知识点链接
- 示例代码片段

**触发条件**
- 代码提交后测试用例未通过
- 用户在AI助手界面选择"帮我分析错误"
- 用户在同一关卡连续3次提交失败（自动触发提示）

**限制**
- 基于预设规则库和常见错误模式匹配
- 对于逻辑错误，仅提供排查方向，不直接给出正确答案
- 不处理系统级错误

#### 12.1.3 必要提示

**功能说明**
在用户主动求助或多次失败时，根据当前练习要求、错误信息和已输入代码，提供与当前问题直接相关的知识点提示与排查方向。

**能力范围**
- 当前错误对应的知识点提示卡片
- 针对当前问题的排查方向
- 简短的下一步操作建议

**触发条件**
- 用户点击AI助手中的提示入口
- 用户在同一关卡连续3次提交失败
- 用户在当前练习页面停留超过10分钟

**限制**
- 提示内容必须与当前课程和当前错误直接相关
- 不提供代码补全、快捷输入或整段答案
- 每个练习 session 最多展示5次提示
- 用户可在设置中关闭提示功能

#### 12.1.4 非必要AI能力说明

**功能说明**
为控制MVP复杂度，个性化任务推荐、开放式问答、代码补全等能力均不纳入本阶段AI能力范围。

**能力范围**
- 不提供个性化任务推荐
- 不提供开放式概念问答
- 不提供代码补全
- 不提供跨练习路径建议

**触发条件**
- 无

**限制**
- 相关需求统一延后至非MVP阶段

### 12.2 AI降级方案（MVP）

**无模型降级方案**

当大语言模型服务不可用时，系统切换至预设规则模式：
- 使用预设语法规则进行代码检查
- 建立常见错误与解释模板的映射表
- 为常见错误建立解释模板映射
- 为每个练习预设静态知识点提示卡片

### 12.3 后续迭代AI增强方向（非MVP）

以下能力不纳入MVP交付范围，仅作为后续迭代的增强方向：

- 基于学习记录与知识掌握情况生成个性化练习推荐与学习路径建议
- 支持带上下文记忆的连续问答能力
- 基于用户学习行为生成更精细的知识掌握度评估
- 生成更具针对性的代码示例与复习建议

**增强前提**

在具备大语言模型调用能力时，增强以下能力：
- 需在MVP稳定交付后再评估接入
- 增强能力不得突破课程边界与人工审核规则
- 所有增强结果仍仅作为学习辅助，不替代课程内容本身

### 12.4 AI功能总体约束

- **不替代人工教学**：AI仅作为辅助工具，所有关键知识点和课程内容由人工设计和审核
- **建议仅供参考**：AI提供的错误解释与提示仅供参考，学习者需自主判断
- **不直接修改代码**：AI不自动修改用户代码，仅提供错误原因与提示方向，修改操作由用户执行
- **数据隐私保护**：用户代码和学习记录仅用于学习辅助，不作与学习目标无关的使用
- **调用限制**：MVP阶段设定每日AI调用上限（如每用户50次/天），超出后切换至降级方案

## 13. 社交功能边界

### 13.1 MVP阶段包含的社交能力

1. **好友添加**
- 学习者可通过搜索昵称或邀请码添加好友
   - 好友关系为双向确认，MVP阶段不支持单向关注
   - 好友列表展示好友昵称、头像与当前学习关卡

2. **成果分享**
- 学习者可将自己的通关成绩、获得的徽章、连续学习天数等成果生成成果卡片
- 成果卡片支持保存到本地相册或在平台内展示给好友
- 好友可在平台内动态流中查看彼此的成果展示

3. **排行榜**
   - 系统维护关卡通关速度榜、周积分榜、连续学习天数榜三类榜单
   - 榜单展示前50名学习者，当前用户排名高亮显示
   - 榜单每日0点刷新，基于前一日数据计算排名

### 13.2 明确不在MVP范围的社交功能

以下功能在MVP阶段明确不开发，避免社交功能过度膨胀，确保核心学习体验优先：

- **论坛/讨论区**：不支持课程章节下的公开评论、问答帖子、话题讨论等社区论坛功能
- **私聊/即时通讯**：不支持好友间的一对一私聊、文字消息、表情发送等即时通讯功能
- **群聊/多人聊天室**：不支持组队内的群聊、学习群组、公开聊天室等多人实时交流功能
- **复杂社区功能**：不支持用户主页、粉丝关注体系、内容发布（如博客/动态）、UGC内容创作与传播等复杂社区功能
- **组队挑战**：不支持多人协作编码任务、组队房间、代码合并预览等组队功能
- **实时语音/视频通话**：不支持组队挑战中的语音或视频实时沟通
- **文件传输**：不支持用户间传输图片、代码文件、文档等任何文件

### 13.3 社交功能约束

- 社交功能以学习互动为核心目标，不发展为通用社交平台
- MVP阶段社交功能以基础互动为主，避免复杂度超出可控范围
- 成果分享内容仅限系统生成的学习数据卡片，不支持用户自定义文本输入
- 不接入第三方社交平台，不支持站外分享、站外互动或第三方关系链导入

## 15. MVP范围

MVP阶段聚焦"学习-练习-闯关-反馈"核心闭环，确保毕业设计周期内可完成开发与验证。以下功能必须在MVP阶段实现：

### 15.1 学习者端（移动APP）

| 模块 | MVP必须实现的功能 |
|------|------------------|
| **账户与档案** | 手机号+验证码注册登录；个人资料编辑（昵称、头像）；学习偏好设置（每日目标、难度偏好） |
| **课程学习** | 课程目录浏览；章节图文内容阅读；学习进度记录与同步；章节完成标记与自动解锁 |
| **编码练习** | APP内代码编辑器（HTML/CSS）；实时预览；代码提交与自动评测；本地草稿自动保存 |
| **测验闯关** | 10-15个线性闯关关卡；关卡地图与解锁状态；关卡内学习+练习+测验整合；1-3星通关评定；通关奖励与解锁下一关 |
| **每日挑战** | 每日0点刷新一道限时练习题；连续学习天数记录；基础经验值奖励 |
| **AI辅助** | 错误解释（自动触发+手动唤起）；必要提示（基于当前错误或当前练习步骤）；5秒内响应 |
| **学习进度** | 个人学习数据总览（学习时长、完成关卡数、代码提交次数、连续天数）；基础可视化图表（学习时长趋势、知识点掌握度） |
| **社交互动** | 好友添加、好友学习行为动态、成果分享、排行榜 |
| **奖励成长** | 经验值累计与等级提升；基础成就徽章（控制在10个以内，如"HTML新手""代码达人""坚持不懈"）；连续学习天数 |

### 15.2 管理后台（Web端）

| 模块 | MVP必须实现的功能 |
|------|------------------|
| **课程与章节管理** | 课程单元创建、编辑、上下架；章节内容组织与排序；教学素材上传 |
| **题目与挑战管理** | 练习题创建（选择/编码）；测试用例配置；闯关关卡配置（任务列表、星级标准、奖励规则） |
| **用户与学习记录管理** | 用户列表查看与搜索；用户学习进度与活跃度查看；账号禁用/启用 |
| **内容审核与反馈处理** | 用户昵称/头像的基础审核；题目纠错与内容反馈处理；处理结果记录 |
| **公告与基础运营配置** | 学习公告发布；每日挑战基础配置；展示类运营文案维护 |
| **数据统计看板** | 核心指标展示（注册用户数、日活跃用户、课程完成率）；基础数据报表（用户趋势、课程数据、题目正确率）；CSV导出 |
| **系统基础配置** | 积分规则配置；徽章定义；AI辅助规则；系统操作记录查看 |

### 15.3 AI能力（MVP）

| 能力 | 说明 |
|------|------|
| 错误解释 | 测试用例失败原因分析、修复建议 |
| 必要提示 | 基于当前错误或当前练习步骤提供知识点提示与排查方向 |
| 无模型降级 | 模板化错误解释、静态知识点卡片与规则化提示 |

### 15.4 明确包含的社交功能（MVP）

| 功能 | 说明 |
|------|------|
| 好友添加 | 搜索昵称/邀请码添加好友，双向确认，上限50人 |

## 17. 异常场景

### 17.1 账号异常

**场景描述**：用户账号出现安全风险或状态异常，影响正常登录或使用。

| 触发条件 | 系统响应 | 预期结果 |
|---------|---------|---------|
| 系统检测到账号在异地或非常用设备登录 | 要求进行额外身份确认 | 验证通过后允许登录 |
| 用户连续多次输入错误验证码 | 临时限制再次尝试并提示稍后重试 | 防止异常频繁尝试 |
| 用户账号被管理员禁用 | 登录时提示"账号已被禁用，请联系管理员" | 明确告知用户账号状态 |
| 用户登录状态过期 | 自动跳转至登录页面，提示"登录状态已过期，请重新登录" | 用户重新登录后恢复原有学习进度 |

### 17.2 任务提交异常

**场景描述**：用户在完成练习或挑战时，提交过程中出现各类异常情况。

| 触发条件 | 系统响应 | 预期结果 |
|---------|---------|---------|
| 用户在网络不稳定时提交代码 | 本地保存代码草稿，提示"网络连接异常，请检查网络后重试" | 用户不丢失已编写代码 |
| 用户快速连续点击提交按钮（重复提交） | 首次提交后锁定提交按钮，显示"提交中..." | 防止重复提交 |
| 用户提交的代码运行超时 | 中断代码执行，返回"代码运行超时，请检查是否存在死循环" | 引导用户自查代码逻辑 |
| 用户提交空代码 | 前端拦截并提示"代码内容为空，请编写有效代码后提交" | 减少无效请求 |

### 17.3 挑战结算异常

**场景描述**：闯关挑战或每日挑战在结算评分时出现数据不一致或边界情况。

| 触发条件 | 系统响应 | 预期结果 |
|---------|---------|---------|
| 用户跨天提交挑战 | 以系统统一时间规则判定挑战日期 | 防止用户通过时间差异获得不当收益 |
| 挑战结算数据异常 | 提供"查看详细评分"入口 | 用户可了解评分依据 |
| 用户完成挑战后系统未发放奖励 | 标记为"待补发"，系统自动重试 | 确保用户应得奖励不丢失 |

### 17.4 社交功能异常

**场景描述**：社交互动过程中出现的异常情况。

| 触发条件 | 系统响应 | 预期结果 |
|---------|---------|---------|
| 好友请求发送后，对方超过24小时未响应 | 系统自动取消该请求，并在列表中标记"请求已过期" | 清理无效请求，避免请求列表堆积 |
| 用户尝试添加已达好友上限的用户 | 提示"对方好友列表已满，无法添加" | 明确告知添加失败原因 |
| 用户查看好友动态时网络断开 | 展示本地缓存的动态数据，提示"网络异常，动态可能不是最新" | 保证基础浏览体验，网络恢复后自动刷新 |

### 17.5 内容审核异常

**场景描述**：用户生成内容（如昵称、头像）或系统展示内容出现违规或不当信息。

| 触发条件 | 系统响应 | 预期结果 |
|---------|---------|---------|
| 用户设置包含违规信息的昵称或上传不合规头像 | 系统提示"内容包含违规信息，请修改后重试" | 从源头拦截违规内容 |
| 系统展示的每日挑战题目包含错误答案 | 用户可通过题目页面"反馈问题"按钮提交纠错 | 确保题目质量，及时修复错误内容 |
| 用户昵称或头像超出平台规范 | 系统按基础审核规则拦截或要求修改 | 维护平台基础内容秩序 |
| AI助手返回的内容包含不当建议 | 每条AI回复下方显示"内容仅供参考" | 明确AI内容边界 |

### 17.6 权限控制异常

**场景描述**：用户尝试访问超出自身权限范围的功能或数据。

| 触发条件 | 系统响应 | 预期结果 |
|---------|---------|---------|
| 学习者尝试直接访问管理后台页面 | 系统检测角色权限并阻止访问 | 防止未授权访问 |
| 学习者尝试查看其他用户的详细学习数据 | 系统校验数据归属，返回"无权查看他人数据" | 保护用户隐私 |
| 管理员尝试执行超出统一角色权限的操作（如修改系统底层配置或访问未授权模块） | 系统拦截该操作，提示"该操作超出您的权限范围" | 防止权限滥用，确保统一角色权限边界清晰 |
| 用户通过篡改参数尝试访问未解锁的关卡或课程 | 系统校验解锁状态，未解锁时返回"该内容尚未解锁" | 防止绕过正常学习路径 |

### 17.7 组队协作边界与作弊防范

**场景描述**：用户尝试访问尚未开放的组队协作功能，或通过异常操作获取不当收益。

| 触发条件 | 系统响应 | 预期结果 |
|---------|---------|---------|
| 用户尝试访问组队协作（组队挑战）入口 | 系统提示"组队协作功能暂未开放，敬请期待" | 明确告知组队协作尚未开放，避免用户困惑 |
| 用户通过篡改请求参数尝试重复领取奖励 | 系统校验奖励发放记录，拦截重复请求 | 防止异常刷取经验值或徽章 |
| 用户利用脚本或自动化工具高频提交代码 | 系统检测异常提交频率，临时限制该账号提交并提示"操作过于频繁，请稍后重试" | 维护公平的学习环境 |
| 用户尝试篡改排行榜数据或伪造通关记录 | 系统校验数据一致性，异常记录不予入库并标记审计 | 保证排行榜与成就数据的真实性 |

### 17.8 通用系统异常

| 触发条件 | 系统响应 | 预期结果 |
|---------|---------|---------|
| 网络中断或弱网环境 | 本地缓存已加载内容，提示用户检查网络 | 保证用户体验连续性 |
| AI服务暂时不可用 | 提示"AI助手暂时离线，可稍后重试或查看文档" | 提供备选方案 |
| 内容加载失败 | 显示占位图和"加载失败"提示，提供"重新加载"按钮 | 用户可主动重试 |
| 服务器内部错误 | 显示"系统繁忙，请稍后重试" | 友好提示用户，避免暴露技术细节 |

| AI必要提示 | 学习者在当前练习卡住时请求提示，系统返回与当前问题相关的知识点提示与排查方向 | 功能验证：在练习页触发提示入口，确认提示内容与当前问题直接相关 |
| 学习进度 | "我的"页面展示学习时长、完成关卡数、代码提交次数、连续天数等数据 | 功能验证：完成学习行为后，确认个人数据总览准确展示 |

### 18.3 非功能验收标准

| 验收项 | 验收条件 | 检查方式 |
|--------|---------|---------|
| APP冷启动 | 从点击图标到首页可交互时间不超过3秒 | 性能测试：使用性能工具测量5次取平均值 |
| 页面切换 | 常规页面切换响应时间不超过500毫秒 | 性能测试：使用性能工具测量10次取平均值 |
| 代码运行 | 代码提交到返回评测结果不超过2秒 | 性能测试：提交标准测试代码，测量5次取平均值 |
| AI响应 | AI单轮问答从发送到返回首条回复不超过5秒 | 性能测试：发送标准问题，测量5次取平均值 |
| 屏幕适配 | APP在320px-430px宽度的手机上布局正常，无元素重叠或截断 | 兼容性测试：在至少3款不同尺寸真机或模拟器上验证 |
| 离线浏览 | 已加载的课程章节在断网后仍可正常阅读 | 手动测试：加载章节后关闭网络，确认内容可继续阅读 |
| 断点续做 | 编码练习中退出APP，重新进入后自动恢复上次代码 | 手动测试：编写代码后强退APP，重新进入练习确认代码恢复 |
| 安全测试 | 未登录用户无法访问需认证页面；普通用户无法访问管理后台 | 安全测试：直接访问受保护接口/页面，确认返回权限错误 |
| 数据隔离 | 用户A无法获取用户B的学习记录或个人信息 | 安全测试：使用用户A的登录状态访问用户B的数据，确认被拒绝 |
