# Draft: Frontend Learning App Refinement

## Requirements (confirmed)
- 项目旨在开发一个面向前端初学者的移动学习APP，将 HTML、CSS 与 JavaScript 的学习过程设计为闯关冒险任务。
- 用户通过完成每日挑战、答题闯关和实践编码任务来解锁关卡与奖励。
- 系统整合必要的AI辅助功能，AI仅用于错误分析与必要提示，不提供代码补全、个性化任务推荐等扩展能力。
- APP支持平台内部的基础社交互动，仅保留好友、学习动态、成果展示与排行榜，不接入第三方平台，也不包含组队协作挑战。
- 学习移动端教育APP开发、前端教学方法、游戏化学习理论和 AI 辅助教学的基本理论与技术，了解国内外行业现状、技术趋势与标准规范。
- 熟悉环境保护和可持续发展的基本理念与相关政策法规，能够在工程实践中从环境与可持续发展的角度评估系统全生命周期中可能产生的风险与影响。
- 在导师指导下阅读国内外文献并自主学习。
- 调研游戏化机制、AI 辅助功能及社交互动策略。
- 设计并开发前端学习移动APP，实现课程学习、编码练习、闯关任务、AI 辅助与好友协作功能，并通过用户测试和学习效果分析验证系统有效性。
- 需要将上述大致需求进一步细化。

## Technical Decisions
- 当前默认交付物为：需求细化 + 模块拆解 + 后续可落地的实施方向（待用户确认是否需要进一步生成正式执行计划）。
- 输出形式已确认：需求分析文档版。
- 当前范围已确认：先只做产品与功能需求，不展开文献调研、行业现状、环保与可持续发展论文内容。
- 参考策略已确认：按全新项目撰写，不参考既有实现作为基线。
- 需求覆盖范围已更新：仅包含学习者端（移动 APP）+ 管理端（Web 即可）。
- 需求深度已确认：按 MVP 核心版撰写，保证毕业设计范围可控。
- 平台边界已更新：学习者端限定为移动 APP；管理端允许使用 Web，不要求移动端后台。
- 后台角色已确认：采用统一后台角色，不再拆分教师端/运营端/管理员三套独立系统。

## Research Findings
- 工作区当前仅确认存在 `.sisyphus/` 规划目录，未发现现成项目代码或业务文档证据。
- 工程与验证基础设施检查结果：无 package manifest、无移动端框架、无后端框架、无测试配置、无 CI、无 lint/format 配置、无 git 仓库。
- 当前应按“从零开始”的项目来细化需求，不能假设已有脚手架、原型或实现基础。
- 但在同级目录 `/home/ltc/graduation_project/` 发现一个高度相关的既有项目 **CodeAdventure**，与当前描述几乎同题：前端初学者、闯关学习、AI 辅助、社交互动、移动端学习 APP。
- CodeAdventure 已具备前端（React Native + Expo）、后端（FastAPI）、管理后台、AI 服务、社交模块及完整软件工程文档，可作为需求细化与模块拆分的重要参考样本。
- 因此当前存在两条合理路径：A) 按“全新项目”细化；B) 按“已有同类项目/毕业设计参考模板”细化并做增强版需求整理。
