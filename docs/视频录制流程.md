### 环境准备

**Mobile Web 分辨率设置**：

Mobile Web 设计分辨率为 400x890（iPhone SE 尺寸），需要在浏览器中模拟：

1. 打开 Chrome 浏览器，访问 `http://localhost:8088`
2. 按 F12 打开开发者工具
3. 点击左上角「切换设备工具栏」图标（或按 Ctrl+Shift+M）
4. 在顶部工具栏选择「iPhone SE」或手动输入尺寸：宽 400，高 890
5. 刷新页面，确保 Mobile Web 在 400x890 分辨率下正常显示

**启动服务**：

```bash
# 终端 1：管理端
cd /home/ltc/CodeQuest/admin && npm run dev          # http://localhost:3000

# 终端 2：Mobile Web
cd /home/ltc/CodeQuest/mobile && flutter build web --release
cd build/web && python3 -m http.server 8088           # http://localhost:8088
```

**✅ 无需启动后端**
- 所有 API 调用失败时自动回退到 Mock 数据，演示效果完整
- 编程题提交会显示模拟判题结果（根据答案正确性显示通过/失败）
- 单选题练习通过 exerciseId 自动识别并加载 Mock 题目
- 社交页面（动态/好友/排行榜）API 失败时展示丰富的 Mock 数据

## AI 帮助功能说明

- **触发位置**：
  - 练习页面底部常驻「AI 帮助」按钮（灰色边框按钮，位于「提交」按钮左侧）
  - 提交答案后，「提交结果」弹窗底部有「查看 AI 帮助」按钮（灰色边框按钮，位于「继续」按钮上方）
- **功能特性**：
  - AI 提供方向性提示，不会直接给出答案或暴露隐藏测试用例
  - 支持多种请求类型：错误解释、解题提示、错误定位、修正建议、操作建议
  - **代码感知智能提示**：系统会分析用户代码，检测CSS选择器、属性、值，识别常见错误（缺少选择器、属性、值、花括号、分号）
- **AI帮助内容示例**：
  - 空代码时：「📝 代码编辑区为空，请先编写代码」
  - 缺少选择器：「❌ 缺少CSS选择器，请添加类选择器(.class)、ID选择器(#id)或标签选择器」
  - 缺少属性：「❌ 缺少CSS属性，请添加background-color、color、margin等属性」
  - 代码完整时：「📝 操作建议，执行步骤：1. 对照测试用例要求，逐个检查代码...」
- **录制建议**：
  - 演示时建议展示AI帮助的三种状态变化（空代码→不完整代码→完整代码）
  - 可强调 AI 的代码感知能力："AI 会分析你的代码，给出针对性的修改建议"
